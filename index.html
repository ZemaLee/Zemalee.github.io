병행 프로세스

두 개 이상의 프로세스가 동시에 처리하는 것을 의미. 배타 접근 보장X -> 커다란 문제 야기

해결책

임계영역 | 상호배제 | 세마포어 | 모니터

  • 임계영역(Critical Section)

    공유하는 자원에 대해 특정 시점에서는 한 프로세스만 사용하도록 허용된 영역

  • 상호배제(Mutual Exclusion)

    여러 개의 프로세스가 공유 자원에 접근할 때, 한 순간에 한 프로세스만 변경할 수 있도록 하는 기법

    (Dekker 알고리즘, Peterson 알고리즘, 인터럽트 비활성화 등의 방법으로 회피 가능)

  • 세마포어(Semaphore)

    Dijkstra에 의해 고안됨.

    보호 변수(Wait / Signal)의 두 가지 연산에 의해서만 Access되도록 통제함.

  • 모니터(Monitor)

    다중 프로그래밍에서 상호 배제를 구현하기 위한 특수한 프로그래밍 기법. 병행성 구조

교착상태

둘 이상의 프로세스가 서로 다른 프로세스가 이미 차지하고있는 자원을 요구해서 영원히 프로세스가 기다리는 현상

교착상태 발생조건

상호배제 | 점유 및 대기 | 비선점 | 환형대기

  • 상호배제

    한 번에 하나의 프로세스만 공유 자원을 사용할 수 있는 상태

  • 점유 및 대기

    프로세스들이 현재 자원을 점유하면서 다른 자원을 요구하는 상태

  • 비선점

    각 프로세스에 할당된 자원은 사용이 완료될 때까지 강제로 해제할 수 없는 상태

  • 환형대기

    서로 다른 프로세스 간 자원요구가 연속적으로 반복되는 상태

교착상태 해결방안

예방 | 회피 | 발견 | 회복

  • 예방

    발생조건을 미리 제거해서 사전에 미리 예방

  • 회피

    발생조건을 제거하지 않고 적절히 피함

  • 발견

    교착상태를 발견하고 발생 원인을 찾아 해결함

  • 회복

    교착상태에 빠진 프로세스를 재시작하거나 원래상태로 돌려서 해결

스케쥴링 알고리즘(잘 모르는거 위주로)

SRT : 남은 작업 추정이 젤 작은거

마감시간(Deadline) : 주어진 제한시간 내에 프로세스가 반드시 완료되도록 하는 방식

MLQ(Multi-Level Queue) : 서로 다른 작업을 각각의 큐에 Time-Slice에 의해 처리

(각 Queue가 독자적 스케쥴링 알고리즘을 사용함)

응집도(Cohesion)와 결합도(Coupling)

응집도는 High, 결합도는 Low가 좋아욤><

  • 응집도 높아야좋음!!

    모듈 내부가 얼마나 강한 연관성으로 뭉쳐있는가를 나타내는 측정치.

    응집도가 증가하도록 설계하는 것이 바람직함.(외부와의 상호교류가 최소화)

  • 결합도 낮아야좋음!!

    모듈 사이의 상호연관성의 복잡도에 대한 측정치

    의존이 많을수록 결합도가 높아진다.

    결합도는 낮게 설계하는 것이 바람직함.

응집도의 종류... 같은 내용도 있는데 pass하겠음..

소프트웨어 3R

역공학(Reverse-Engineering) | 재공학(Re-Engineering) | 재사용(Re-use)

역공학

구현된 소프트웨어를 역으로 추적해 설계기법 등의 자료를 얻어내는 일.

  • 역공학의 장점
    • 상용화되거나 이미 개발된 소프트웨어 분석을 도움
    • 기존 시스템의 자료와 정보를 설계 수준에서 분석할 수 있기 때문에 유지/보수성을 향상시킴

재공학

유지/보수 비용을 감소시키기 위한 방법. 소프트웨어를 새 형태로 재구성하기 위한 검사/변경/새형태로의구현

  • 재공학의 장점
    • 복잡한 시스템을 다루는 방법:자동화도구(CASE)
    • Side-effect 발견
    • 재사용 용이
    • 고수준 추상화 합성

재사용

걍 진짜 갖다쓰는거

  • 재사용의 장점
    • 개발기간 감소
    • 에러 방지
    • 이거 정처기때 나온거네

클라우드 컴퓨팅

컴퓨터 용량을 효율적으로 활용하고 불확실한 서비스의 수요에 유연하게 대처하기 위한 목적으로 개발

클라우드 서비스 도입 기대효과

  • 비용
    • 운영비용은 증가하지만, 미래의 이윤을 생각했을 때 전체적 비용 감소
  • 기간
    • 개발 기간 단축, 제품 개발주기 단축
  • 운영
    • 운영 인력 감소, 자원 효율성 강화
  • 제품
    • 제품 집중도 향상

다른 컴퓨팅과의 비교

그리드 컴퓨팅

그리드 컴퓨팅 : "네트워크 상에서 컴퓨터나 데이터 등의 리소스를 가상화하여 통합하고, 필요에 의해 가상 컴퓨터를 동적으로 생성하기 위한 구조"

  • 클라우드 컴퓨팅과의 유사점

    • 분산 컴퓨팅 구조 사용
    • 가상화된 컴퓨팅 자원 제공

  • 클라우드 컴퓨팅과의 차이점

    • 그리드 : 인터넷 상의 모든 컴퓨터 자원 사용
    • 클라우드 : 사업자 사유 컴퓨팅만 사용

(참고) 그리드 컴퓨팅은 과학 기술적 계산, 대규모 연산 등 병렬성이 높은 Application에 사용

(참고) 클라우드 컴퓨팅은 웹App 등 범용적인 용도로 사용

유틸리티 컴퓨팅

유틸리티 컴퓨팅 : 컴퓨팅 자원을 사용하는 만큼 돈을 지불하는 형태로 컴퓨팅 자원을 제공하는 방식

클라우드 컴퓨팅 유형

서비스 유형에 따른 분류

IaaS | PaaS | SaaS

IaaS(Infrastructure as a Service)

인터넷 네트워크를 경유하여 인프라 자원(서버, 스토리지, 네트워크)를 서비스로 제공하는 형태

IaaS 사용자는 운영체제 이상의 영역을 직접 관리해야 한다.

사용자 입장에서는 기존 서비스 업체에서 서버를 호스팅 받는것과 유사하다.

PaaS(Platform as a Service)

개발 및 운영 환경을 구축할 필요 없이 필요한 개발 및 운영 환경을 서비스 형태로 사용하는 방식

서비스 범위 ~ 개발환경 | 서비스배포 | 운영환경

PaaS 사용자는 앱, 데이터를 직접 관리해야 한다.

ex) DB로 MySQL 쓰는 자바 웹앱 개발시, MySQL, Apache Tomcat 제공하는 서비스 이용해서 신속하게 개발하는 것이 가능함.

SaaS(Software as a Service)

소프트웨어를 제공하는 서비스로, 소프트웨어를 웹브라우저 등 클라이언트를 통해 사용자에게 전달하는 모델

SaaS 사용자는 소프트웨어가 어디 있는지, 어떤 OS를 쓰는지, 어떤 언어로 개발되었는지 알 필요 없이 필요한 기능을 사용하면 된다.

ex) 구글 Docs

  • SaaS 서비스 특징
    • 웹 브라우저로 접근(설치형X 웹브라우저로 접근)
    • 사용량 기반 비용(소프트웨어 사용량만큼 비용 지불)
    • On-Demand(필요한 소프트웨어는 즉시 사용 가능)
    • IT 수요 최적화(관리/확장 이슈가 없음)
